# Support Connect 🚀

A real-time video-assisted customer support platform built for the modern web. Support Connect allows customer service agents to conduct high-quality, low-latency video calls with customers directly through their browser—no app installation required!

**🔴 Live Demo:** [https://support-connect.vercel.app/](https://support-connect.vercel.app/)

## 🌟 Features

- **Agent Dashboard:** Secure login and session management for support agents.
- **One-Click Joining:** Automatically generate unique invite links (via `nanoid`) that customers can click to instantly join a support room.
- **Real-Time Video (WebRTC):** Powered by an AWS EC2-hosted LiveKit media server, ensuring high-quality, ultra-low latency video and audio streams.
- **Smart Layouts:** Utilizes `@livekit/components-react` to automatically detect active speakers and adjust grid layouts dynamically.
- **Live Chat Sidebar:** Custom, real-time messaging system built on top of Socket.io allowing seamless text communication during video calls.
- **Metrics & Analytics:** Integrated Prometheus-compatible metrics endpoint for tracking active sessions and connection durations.

## 🏗 Architecture

Support Connect utilizes a secure 3-Tier Architecture:

1. **Frontend:** React 19, TanStack Router, Zustand (State), Tailwind CSS v4, and Shadcn UI.
2. **Backend API:** Node.js, Express.js, Socket.io (WebSocket handling), and LiveKit Server SDK.
3. **Database & Auth:** Supabase (PostgreSQL) handling user authentication (JWTs) and persistent storage for sessions, messages, and participants.

## 🛠 Prerequisites

Before setting up the project locally, ensure you have:
- Node.js (v22+)
- A [Supabase](https://supabase.com/) account and project.
- A running [LiveKit Server](https://docs.livekit.io/realtime/server/deployment/) (e.g., hosted on AWS EC2).

## ⚙️ Setup Instructions

### 1. Database Setup (Supabase)

1. Create a new Supabase project.
2. Open the Supabase SQL Editor and execute the following schema to generate the required tables:

```sql
CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_by UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  invite_token TEXT UNIQUE NOT NULL,
  status TEXT DEFAULT 'active' CHECK (status IN ('active', 'ended')),
  started_at TIMESTAMPTZ DEFAULT now(),
  ended_at TIMESTAMPTZ,
  duration_seconds INTEGER,
  livekit_room_name TEXT NOT NULL
);

CREATE TABLE participants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('agent', 'customer')),
  joined_at TIMESTAMPTZ DEFAULT now(),
  left_at TIMESTAMPTZ,
  socket_id TEXT
);

CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
  sender_name TEXT NOT NULL,
  content TEXT NOT NULL,
  sent_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE recordings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
  status TEXT CHECK (status IN ('processing', 'ready')),
  file_url TEXT,
  started_at TIMESTAMPTZ,
  ended_at TIMESTAMPTZ
);
```

3. In Supabase Authentication settings, ensure **Email Confirmations** are disabled so agents can log in immediately after registration.

### 2. Backend Configuration

1. Navigate to the `backend/` directory:
   ```bash
   cd backend
   npm install
   ```
2. Create a `.env` file in the `backend/` folder with the following variables:
   ```env
   SUPABASE_URL=your_supabase_project_url
   SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
   LIVEKIT_HOST=http://your_ec2_ip:7880
   LIVEKIT_API_KEY=your_livekit_api_key
   LIVEKIT_API_SECRET=your_livekit_api_secret
   ```
3. Start the backend development server:
   ```bash
   npm run dev
   ```
   *(The server will run on `http://localhost:8000`)*

### 3. Frontend Configuration

1. Navigate to the `frontend/` directory:
   ```bash
   cd frontend
   npm install
   ```
   *(Note: Vite is configured to proxy all `/api` requests to `http://localhost:8000` to prevent CORS issues).*

2. Start the frontend development server:
   ```bash
   npm run dev
   ```

### 4. Running the App

1. Open `http://localhost:5173/login` in your browser.
2. Register a new Agent account.
3. Once in the Dashboard, click **New Session** to generate a room.
4. Copy the invite link and open it in an Incognito window to join as a Customer.
5. Enjoy your ultra-low latency WebRTC video call!

## 🔒 Security Notes

- The frontend communicates securely with the backend API.
- The backend communicates with Supabase using the `SERVICE_ROLE_KEY` to bypass restrictive RLS policies, as the backend acts as the sole source of truth and validation.
- LiveKit WebRTC tokens are dynamically generated by the backend and have strict `roomJoin` and scoped permissions.
